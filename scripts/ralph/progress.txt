# Progress Log — KithNode MVP

## Codebase Patterns
- Next.js 16 removed `next lint` CLI command — use `eslint src` directly
- Tailwind CSS v4 uses `@import "tailwindcss"` instead of `@tailwind` directives
- Tailwind v4 PostCSS plugin is `@tailwindcss/postcss` (not `tailwindcss`)
- ESLint 9 flat config — FlatCompat has circular reference issues with eslint-config-next; use `@eslint/js` + `@typescript-eslint/parser` directly
- Vitest v4 works well with `@vitejs/plugin-react` and `jsdom` environment
- `create-next-app` won't run in a directory with existing files — initialize manually
- Prisma v7 uses `prisma-client` generator (not `prisma-client-js`) — generates ESM-only TypeScript
- Prisma v7 generated client requires a driver adapter (e.g., `@prisma/adapter-better-sqlite3`) — `new PrismaClient()` without adapter throws
- Prisma v7 generated client can't be run standalone with tsx CJS mode — use dynamic `import()` in seed scripts
- The adapter class is `PrismaBetterSqlite3` (lowercase "qlite"), not `PrismaBetterSQLite3`
- DATABASE_URL `file:./dev.db` creates db at project root (not prisma/); keep paths consistent
- Must add `src/generated/` to ESLint ignores to avoid linting Prisma generated code
