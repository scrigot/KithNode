## Codebase Patterns
- Server components by default; client components only for interactivity ("use client")
- Auth: import { auth } from "@/lib/auth" in server components, check session
- DB: import { prisma } from "@/lib/db" — singleton PrismaClient with BetterSqlite3 adapter
- Tests: Vitest + @testing-library/react, mock prisma and auth before importing modules
- ESLint: flat config (eslint.config.mjs), globals for browser/node APIs added as needed
- API routes: NextResponse.json() for responses, auth() for session check

## Gotchas
- ESLint config needs explicit globals for fetch, document, global (no env presets)
- Tests need afterEach(cleanup) to avoid multiple render accumulation with @testing-library/react
- Prisma uses BetterSqlite3 adapter — import from "../generated/prisma/client.js"
